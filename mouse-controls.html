Mouse-only control design, with the right technical language

The whole game can be controlled with the mouse. The player slides left and right before the throw, holds the left mouse button to enter an aiming/charging gesture, drags upward for power, drags left or right for direction, releases to launch, then lightly steers the ball with mouse movement while it travels down the lane.

Mouse onlyUE5 input lingoFuture ChatGPT ready
Mouse gesture diagramPosition, charge, aim, release, steer

Think of the controls as a small input state machine. Before the throw, mouse X controls the player lane position. While the left button is held, the drag vector controls power and aim yaw. After release, mouse X becomes a light steering input.

PositioningMouse left/right maps to lane-space Y before the throw.
Hold + dragUp sets power. Left/right sets throw direction.
ReleaseBall launches with selected power and yaw.
SteeringAfter launch, mouse X applies a small lateral force.
Pre-throw movementMouseDeltaX -> DesiredLaneY
Move the player/ball spawn left and right across a clamped lane range.
Power gesturePower01 = Clamp(DragUpPixels / MaxPowerDragPixels, 0, 1)
Dragging upward farther gives more launch speed.
Aim directionAimYawDegrees = Clamp(DragRightPixels / MaxAimDragPixels, -MaxYaw, MaxYaw)
Left/right while held creates the slash direction: \ or /.
LaunchForwardVector.RotateYaw(AimYaw) * LaunchSpeed
Release left mouse to apply impulse or set velocity.
Post-release steeringSteerInput = Clamp(MouseDeltaX / SteerSensitivity, -1, 1)
Apply a weak lateral force/torque, much weaker than the launch impulse.

Plain English control loop

  1. Move the mouse left or right to choose where the player stands on the approach.
  2. Press and hold the left mouse button to start the throw gesture.
  3. Drag upward to increase power.
  4. While still holding, drag left or right to angle the throw.
  5. Release the left mouse button to launch the ball.
  6. After release, keep moving the mouse left or right to gently steer the ball.

Technical version

This is a mouse-only lane-space throw mechanic using a pre-launch lateral positioning state, a click-and-drag charge/aim gesture, a launch impulse, and a post-launch steering assist. The important variables are player lane Y, drag vector, normalized power, aim yaw, launch speed, steering input, and steering force.

Input state machineUse these names in Blueprint or C++

These state names are the lingo that will help ChatGPT understand what you want later. Each state has one job, which keeps the control code easier to build and debug.

PositioningNo button held. Mouse X moves the player/ball spawn left and right across the approach.
ChargingLeft mouse button is held. Store DragStartScreenPos and capture the cursor.
AimingUse current drag vector to calculate Power01 and AimYawDegrees.
LaunchOn mouse release, convert power and yaw into a forward launch vector.
SteeringBall is active. Mouse X applies weak lateral force or torque.
ResultWhen the ball reaches pit or stops, score pins and reset for the next throw.

Blueprint/C++ event wording

Control event flow
OnMouseMove:
  if State == Positioning:
    DesiredLaneY += MouseDeltaX * PositionSensitivity
    DesiredLaneY = Clamp(DesiredLaneY, MinLaneY, MaxLaneY)

  if State == Steering:
    SteerInput = Clamp(MouseDeltaX / SteerPixelsForFullInput, -1, 1)

OnLeftMouseDown:
  State = Charging
  DragStartScreenPos = CurrentMouseScreenPos
  CaptureMouse()

OnMouseDragWhileHeld:
  CurrentDragVector = CurrentMouseScreenPos - DragStartScreenPos
  Power01 = Clamp(-CurrentDragVector.Y / MaxPowerDragPixels, 0, 1)
  AimYawDegrees = Clamp(CurrentDragVector.X / MaxAimDragPixels, -1, 1) * MaxAimYawDegrees
  State = Aiming

OnLeftMouseUp:
  LaunchDirection = LaneForwardVector rotated by AimYawDegrees
  LaunchSpeed = Lerp(LaunchSpeedMin, LaunchSpeedMax, Power01)
  LaunchBall(LaunchDirection, LaunchSpeed)
  State = Steering

Physics wording

  • Impulse: one strong push at launch.
  • Velocity: directly setting ball movement speed.
  • Lateral force: gentle side push after launch.
  • Torque: spin-based steering, more physical but harder to tune.

Tuning safeguards

  • Clamp lane position to allowed Y range.
  • Add a small deadzone to drag and steering.
  • Limit maximum aim yaw.
  • Make post-release steering subtle.
  • Disable steering after impact or when speed is too low.
