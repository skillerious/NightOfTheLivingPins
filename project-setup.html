Set the project up like a real game from day one

A tidy level is easier to art-direct, easier to hook up to C++, and much easier to duplicate for future stages. This folder structure gives you a clean base before you place a single cube.

Exact editor setup clicks

  1. Open the Content Drawer with Ctrl + Space.
  2. Right-click inside Content and create NightOfTheLivingPins.
  3. Open that folder and create the child folders from the list.
  4. Use File > New Level, choose an empty level, then save it into Maps as L_BowlingAlley_Template.
  5. Use File > Save All once the folder structure and map exist.

Outliner folder workflow

  1. Open Window > Outliner if the Outliner is missing.
  2. Create each numbered folder in the Outliner before adding actors.
  3. After placing any actor, drag it into the matching folder immediately.
  4. Use the Outliner search box to find actors by prefix: SM_, TP_, TV_, RectLight_, or BP_.
Beginner rule: every actor you add should go into a folder immediately. If you keep the Outliner tidy now, future Robin will thank you when C++ spawners, triggers, cameras, and VFX start arriving.
Save habit: save the level after each major pass. Use names like L_BowlingAlley_Template, then duplicate later to L_Lane13_Prototype so the clean template is never damaged.
Project structure pipelineHow the folders support gameplay

Build the level as a chain of clean containers. Each stage feeds the next one, so art, collision, markers, and code hooks do not become tangled.

Content foldersMaps, Blueprints, Materials, Meshes, FX, Audio, UI, and Dev notes stay separated.
Template mapL_BowlingAlley_Template stores the clean reusable shell before one-off gameplay is added.
Outliner foldersBlockout, lighting, props, triggers, markers, cameras, FX, and audio are grouped in-scene.
Named actorsEvery important object gets a stable prefix: SM_, BP_, TP_, TV_, LI_.
C++ accessSpawner and manager classes can find tagged actors without hard-coded mystery coordinates.
